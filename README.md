
# Agile Software Project Tutorial

A concise, phase-by-phase guide for running an Agile software project. This repository presents each project phase, recommended practices, and links to supporting video tutorials and artifacts.

## What you'll learn

- The main phases of an Agile project and their goals.
- Practical practices to improve code quality, testing, and delivery.
- Techniques to strengthen team collaboration and Agile processes.

Tutorial videos are available in the [Self-Directed Training collection](https://github.com/retamotome/SelfDirectedTraining). 

## How to use this repository

1. Browse phase folders in sequence to follow the recommended workflow.
2. Read the short tutorials and apply provided templates or checklists.
3. Watch the linked videos for guided walkthroughs and examples.

## Overview of the tutorial

This tutorial covers software engineering activities including development, testing, and maintenance. For an introductory video, see [Lesson 01: Agile and Software Engineering](https://github.com/retamotome/SelfDirectedTraining/blob/main/AgileSoftwareEngineering/2025-11-09_AgileProjectMgt/README.md). 

Key phases included:

- Project Initiation
  - Project planning, team formation, and project setup.
  - Goal: define vision, roles, and a high-level roadmap.
  - Companion video: [Agile Team — Identification and Management](https://github.com/retamotome/SelfDirectedTraining/blob/main/AgileSoftwareEngineering/2026-02-22_AgileTeam/README.md)   
  - 🏆[Agile Code Conduct 敏捷團隊行為守則](./AgileCodeConduct/README.md)

- [Business Development](./BusinessDevelopmentPhase/README.md)
  - Market research, business case development, and stakeholder engagement.  
  - Business Requirements Dedinition  
	- Elicit, document, and validate stakeholder requirements.  

- Software Development
  - Iterative coding, testing, and integration to deliver working software.

- Maintenance
  - Bug fixes, performance improvements, and feature updates after deployment.
